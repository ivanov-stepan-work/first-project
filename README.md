# Система формирования отчётов по индивидуальным проектам студентов

Учебно-прикладной веб-проект для автоматизации формирования отчётов по индивидуальным проектам студентов.  
Проект разрабатывался как первый самостоятельный опыт создания полноценного веб-приложения: от анализа предметной области до реализации и поддержки.

## Что делает проект
- хранение информации об индивидуальных проектах студентов
- формирование отчётов на основе введённых данных
- базовая система авторизации пользователей
- администрирование и редактирование данных

## Моя роль
- анализ предметной области
- проектирование структуры базы данных
- разработка бэкенда на PHP
- разработка фронтенда (PHP + JavaScript / jQuery)
- реализация авторизации
- последующая поддержка проекта (добавление функционала, исправление ошибок)

## Стек
- Backend: PHP
- Frontend: PHP, JavaScript, jQuery
- Database: SQL (проектирование БД)
- Прочее: HTML, CSS


## Запуск проекта (локально)
Проект учебный и не предназначен для production-деплоя.

Минимальные требования:
- PHP
- Web-сервер (Apache / Nginx)
- SQL-совместимая БД

1) Склонировать репозиторий  
2) Создать базу данных и импортировать структуру (см. `/db`)  
3) Скопировать `.env.example` → `.env` и указать параметры подключения к БД  
4) Запустить проект через локальный сервер

## Ограничения и возможные улучшения
Проект создавался как первый самостоятельный учебно-прикладной опыт, поэтому сейчас вижу несколько точек роста:

- Есть повторяющиеся фрагменты кода — логичнее вынести общие части в функции/классы и переиспользуемые модули.
- Обмен данными реализован в классическом формате с перезагрузкой страниц. Как улучшение, можно добавить AJAX-запросы (fetch/XHR) для части операций (создание/редактирование/поиск), чтобы повысить отзывчивость интерфейса.
- В перспективе проект можно привести к более структурной архитектуре (например, разделить логику, шаблоны и доступ к данным), а также добавить более строгую валидацию входных данных.
